# ISO 8601 Date Text Inputs

How to make date entry display and store as ISO 8601 (`YYYY-MM-DD`) consistently, regardless of the
user's locale. This is the pattern used across Melia frontends (e.g. `melia-misc/pto`, `melia-schwab`).

## Why not `<input type="date">`

The native date input is tempting (free calendar picker) but it has a fatal flaw for ISO display:

- The **displayed/typed format follows the user's OS/browser locale** — US users see `MM/DD/YYYY`.
  You cannot change this with CSS.
- `lang="en-CA"` on the input (en-CA formats as ISO) **does not reliably override it in Chrome** —
  Chrome uses the OS locale for the date field, not the element's `lang`. Tested: still shows US.

The `.value` of `type="date"` *is* ISO, but the **display** can't be forced to ISO. Since users need to
read/enter dates in ISO, use a plain text input instead.

## The pattern: masked, validated text input

```html
<input type="text" inputmode="numeric" maxlength="10" placeholder="YYYY-MM-DD"
       oninput="this.value=maskIsoDate(this.value);onDateChanged()">
```

- `type="text"` → display is exactly what we put in the value (ISO), locale-independent.
- `inputmode="numeric"` → mobile numeric keypad.
- `maxlength="10"` → `YYYY-MM-DD` is 10 chars.
- The element's `.value` is the ISO string — store/send it as-is. No parsing/formatting on submit.

### Mask: digits-only, auto-dash, max 8 digits

Strips non-digits, caps at 8 digits, and inserts the dashes in the right places as the user types
(`20260715` → `2026-07-15`). Backspacing through a dash removes it naturally.

```js
function maskIsoDate(v){
  var d=(v||'').replace(/\D/g,'').slice(0,8);
  var out=d.slice(0,4);
  if(d.length>4) out+='-'+d.slice(4,6);
  if(d.length>6) out+='-'+d.slice(6,8);
  return out;
}
```

### Validate: real calendar date

Format alone isn't enough — reject impossible dates (`2026-02-29`, `2026-13-01`, `2026-04-31`).
Year range 1900–2100, month 01–12, day per month with **leap-year awareness**
(`new Date(Date.UTC(y, m, 0)).getUTCDate()` gives the day count for 1-based month `m`).

```js
function isValidIsoDate(str){
  if(!/^\d{4}-\d{2}-\d{2}$/.test(str)) return false;
  var y=+str.slice(0,4), m=+str.slice(5,7), d=+str.slice(8,10);
  if(y<1900||y>2100||m<1||m>12) return false;
  var dim=new Date(Date.UTC(y,m,0)).getUTCDate();
  return d>=1 && d<=dim;
}
```

Validate on the client (block submit with a clear message) **and** on the server (enforcement) — the
same function ports verbatim to Node. Don't trust the client.

## Turning a date into an instant (timezone-aware)

When a date needs to become an absolute instant (e.g. EventBridge `StartDate`/`EndDate`, "start of day"
vs "end of day" in a specific timezone), convert the wall-clock date in the target tz to UTC. This is
DST-safe and uses only `Intl` (no libraries):

```js
function tzOffsetMs(tz, instantMs){
  var p={};
  new Intl.DateTimeFormat('en-US',{timeZone:tz,year:'numeric',month:'2-digit',day:'2-digit',
    hour:'2-digit',minute:'2-digit',second:'2-digit',hour12:false})
    .formatToParts(new Date(instantMs)).forEach(function(x){p[x.type]=x.value;});
  var hr=p.hour==='24'?0:+p.hour;
  return Date.UTC(+p.year,+p.month-1,+p.day,hr,+p.minute,+p.second)-instantMs;
}
function zonedToInstant(tz,y,mo,d,h,mi,se){
  var guess=Date.UTC(y,mo-1,d,h,mi,se||0);
  var off=tzOffsetMs(tz,guess); var inst=guess-off;
  var off2=tzOffsetMs(tz,inst); if(off2!==off) inst=guess-off2; // refine across a DST boundary
  return inst;
}
// start of day: zonedToInstant(tz,y,mo,d,0,0,0)   end of day: zonedToInstant(tz,y,mo,d,23,59,59)
```

## Producing ISO strings in JS

`en-CA` IS reliable for *formatting* (the locale renders dates as ISO). Use it with `Intl` to get a
`YYYY-MM-DD` string for a given timezone:

```js
function ymdInTz(tz, date){
  return new Intl.DateTimeFormat('en-CA',{timeZone:tz,year:'numeric',month:'2-digit',day:'2-digit'})
    .format(date); // e.g. "2026-07-15"
}
```

## Summary

| Need | Do |
|------|----|
| Display/enter ISO date | `type="text"` + `maskIsoDate` on `oninput` (NOT `type="date"`) |
| Reject bad dates | `isValidIsoDate` on client **and** server |
| Date → instant in a tz | `zonedToInstant` (Intl-based, DST-safe) |
| Format a Date as ISO | `Intl.DateTimeFormat('en-CA', …)` |
